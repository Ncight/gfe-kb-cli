# 1.1 从 YJK 导出的 YDB 文件的必要文件有哪几项？

答：必须项有 dtlmodel.ydb 与 dtlCalc.ydb，这两个文件自动生成于 YJK 计算路径的子文件夹“施工图”中；YJK 上部结构计算完成后，分别打开 YJK 的模型荷载输入与设计结果模块，点击左上方标题栏中带有蓝色箭头的按钮，即可分别导出文件 dtlmodel.ydb 与 dtlCalc.ydb。如下截图所示。

# 1.4 YJK 模型已建模基础，导入 GFE 时点选读取基础模型，但 GFE 模型中的基础与 YJK 不一致是什么原因？

答：首先应明确 YJK 模型中基础建模在哪个模块，若基础建模在模型荷载输入模块，则基础信息储存在 dtlmodel.ydb 文件中，若基础建模在基础设计模块，则基础信息储存在 jccad\_0.ydb 中。

dtlmodel.ydb 的优先级会高于 jccad\_0.ydb，即在模型荷载输入模块中若已建模基础，则 GFE 会直接读取该基础，而不读取基础设计模块中的基础。

# 1.5 YJK 模型已建模基础，导入 GFE 时点选自动生成基础模型，但 GFE 模型中仍导入了 YJK 的桩基础是什么原因？

答：当存在 jccad\_0.ydb 文件时，即使点选“自动生成基础模型”，GFE 仍会读取 jccad 中的桩信息，但不会读取筏板相关的信息，而是自动生成筏板进行替换。（自动生成筏板基础）

# 1.6 当存在多层地下室，且不同层的地下室均有局部基础时，是否可以在 GFE 中导入或自动生成？

答：需先按 1.3 条进行 YJK 模型检查，尤其 “地下室层数”、“与基础相连构件的最大底标高”、“与基础相接的最大楼层号” 必须设置正确。YJK 模型检查无误后，用户可以选择 “读取基础模型” 或 “自动生成基础模型”：

1. 读取基础模型：勾选后，若 YJK 模型文件夹中存在 Jccad\_0.ydb 文件，则接口会读取 Jccad\_0.ydb 的所有信息；但注意仅地下室楼层会导入完整的筏板、承台、桩基、地梁，对于非地下室楼层仅导入桩基；对于地下室未建模基础的楼层，根据“与基础相接的最大楼层号”会自动生成筏板基础；若 YJK 模型未设置地下室，则仅底层会导入完整的筏板、承台、桩基、地梁，其余楼层仅导入桩基。

若不存在 Jccad\_0.ydb，则接口会读取 dtlmodel.ydb 中地下室楼层的基础信息，即 YJK 模型在荷载输入模块建立的地下室基础，对于非地下室楼层的基础信息不读取。若 YJK 模型未设置地下室，则不导入任何基础信息。

若 Jccad\_0.ydb 及 dtlmodel.ydb 均不存在，则导入 GFE 后的模型将无基础。

(dtlmodel.ydb 记录的基础信息往往只适用于非常简单规则的项目，一般情况下建议在 YJK 基础模块中建立基础模型，即将基础相关信息记录在 Jccad\_0.ydb 文件中)

该参数往往需结合“筏板最大周长”、“筏板或承台超出外轮廓的最大距离”两项参数使用，才可确保导入的基础完全正确。

2. 自动生成基础模型：勾选后，若存在 Jccad\_0.ydb 文件，则不读取 YJK 筏板信息，仅读取桩信息及最外围基础投影线的信息。对于层号小于等于 YJK 基础模块参数“与基础相连的最大楼层号”的基础楼层，且属于地下室的楼层，无论用户是否建模筏板，都将自动生成筏板基础。若 YJK 模型未设置地下室，则仅自动生成最底层筏板基础。

若不存在 Jccad\_0.ydb 文件，则 GFE 不再读取 YJK 模型的任何基础信息，对于地下室楼层，无论用户是否建模筏板，都将自动生成筏板。若某个地下室楼层的楼面以下，均存在下一个楼层的天花板，则本楼层不生成筏板。若 YJK 模型未设置地下室，则无法自动生成筏板基础。

该参数往往需结合“自动生成筏板厚度”参数使用，才可确保自动生成的筏板基础完全正确。

# 1.7 YJK 基础导入常见问题及解决方法:

1. YJK 在荷载输入模块建立了基础，与基础模块建立的基础出现了数据冲突。应将荷载输入模块建立的基础构件清除。  
2. YJK 模型的 “地下室层数” 数值设置错误。应将所有期望导入/自动生成基础的楼层都应设为地下室楼层。  
3. YJK 模型的 “与基础相接的最大楼层号” 数值设置错误。应将所有期望导入/自动生成基础的楼层都应设为与基础相接。  
4. 存在各边线均超出地下室外轮廓的筏板，即完全不与地下室外轮廓相交的筏板。应利用筏板减薄区等方式划分筏板轮廓线，使筏板轮廓线与地下室外轮廓线出现相交。  
5. “筏板最大周长” 参数取值太小。应根据实际的筏板轮廓线周长修改。  
6. “筏板或承台超出外轮廓的距离” 参数取值太小。应根据实际的筏板或承台情况进行修改。

# 1.8 YJK 模型导入 GFE 时出现闪退，或一直卡在导入界面无法继续，是什么原因？

答：需先按 1.3 条进行模型检查，尤其 1.3-3 检查地下室外轮廓是否闭合。

一般这种情况有可能是模型中存在大量未正确建模的构件，需注意的是 YJK 模型应避免短墙、构件重叠、上下层墙柱节点不对齐等明显建模错误，或者出现错误的材料编号（如 C35 误输入为 C33 等），楼板太长太异形等，用户应先仔细检查 YJK 模型，确保无误后再导入 GFE。

也有可能是存在大量的弧墙弧梁重叠、或者弧形节点非常多上下层错位等原因，一般可以适当简化弧形为多段折线并上下层对齐进行修正。

更多常见的 YJK 模型的建模问题见 1.16 条。

# 1.10 YJK 模型导入 GFE 后，如何确保模型导入完整？

答：1. 首先可以从表观上对模型完整性进行判断，确认是否有结构构件丢失、基础是否导入成功/自动生成；  
2. YJK 模型导入 GFE 后，会在 YDB 文件路径下会自动生成 prep\_notes.txt 文件，其中内容以 sum 开头，就是总质量（1.0 恒+0.5 活）。可用于与 YJK 计算结果的 wmass.out 文件中的总质量进行对比。

（上述的活荷载组合系数 0.5，是读取了 YJK 模型中的参数“活荷载重力荷载代表值组合系数”。另外注意，水平力及向上的力不转为质量）

(若质量对比存在局部误差，可能是 YJK 中勾选了自重计算时扣除重叠的选项、YJK 荷载转质量是除以 10 而不是 9.8、材料容重不同、基础筏板差异等原因所导致。)

![](../assets/GFE-FAQ/1f60f7bf11fc76d7fd2648e83dbe88d77e3abe2bc39aeec658570d0d19db2daf.jpg)

# 1.11 YJK 模型导入 GFE 后，是否可以记录下导入选项，方便二次导入时可以按上次的选项？

答：YJK 模型导入 GFE 后，会在导入时设置的主目录下自动生成” prop.txt “，里面记录了上一次导入时的导入设置，并且当该文件存在时，后续导入该 YJK 模型都会默认读取该文本，因此二次导入时会默认按上次的导入选项。

# 1.13 YJK 模型导入 GFE 后，有部分梁悬空、板缺失，是什么原因？

答：一般当梁与梁夹角太小，或三角形板角度太小时，接口程序会自动过滤掉该构件，以避免有限元

分析中出现畸形网格的问题。用户可以通过调整 YJK 接口导入参数中的“梁夹角阈值”和“最小板内角”参数进行导入。也有可能是板太异形导致的丢失。

![](../assets/GFE-FAQ/fb37cdd9f76f947527a6efe019096897fe9370c659aaa3682153e6671732dfb0.jpg)

# 1.14 YJK 模型导入 GFE 后，筏板基础有部分板缺失，是什么原因？

答：一般是结构轴线缺失，或结构轴线太复杂导致。用户可以在缺失筏板的位置补齐轴线或调整轴线，使轴线划分的板块看起来尽量规整。

还有一种特殊情况，是地下室底板出现类似平面中部掏空的情况，比如地下二层外轮廓为地下一层外轮廓的子集，目前 YJK 数据接口的多边形布尔运算不支持这种情况。用户可以稍加大地下二层外轮廓，以保证地下二层外轮廓两个或以上的点不为地下一层外轮廓的内部，位于或超出地下一层外轮廓外边线即可。

# 1.15 YJK 模型无大片筏板基础及地下室，仅结构底部有地基梁、独立基础或条形基础等，应该如何导入 GFE 模型？

答：这种情况比较特殊，按目前 YJK 数据接口的逻辑筏板与地基梁、条形基础等同属基础构件，并且接口在导入基础构件时默认会生成筏板。针对此场景对 YJK 模型建模约定按如下区分：

1. 有地下室结构，且有筏板，都在 YJK 基础模块建模及导入。  
2. 无地下室结构，且无大筏板，都在 YJK 荷载输入模块建模及导入。注意所有基础构件的标高，都应位于 YJK 模型的自然层底。

# 1.17 YJK 模型导入后，检查模型的顺序是怎样的？

答：一般按如下顺序检查比较合理：

1. 外轮廓箱体是否正常，如果异常可能导致结构或土体网格划分失败。一般可以通过调整地下室

最外围结构构件的偏心及修复上下层节点错位等方式进行修正。

2. 筏板是否生成正确，有无多余筏板，有无筏板洞。一般需结合 YJK 接口导入参数及 YJK 模型中地下室层数等方式进行修正。  
3. 将不生成基础的模型导入 GFE，统计质量与 YJK 模型质量进行对比，判断是否存在构件丢失或荷载缺失等情况。

# 1.19 YJK 模型的减隔震构件导入后，对于刚度来说读取的是等效刚度还是非线性刚度？

答：GFE 默认读取 YJK 模型中隔震支座及减震器的非线性刚度，但是如果要做模态、静力分析，应该由用户手动将 Connector 连接器的参数改为“等效刚度”。

# 1.20 YJK 模型的隔震支座导入后会变成 Connector 连接器，各参数及物理量对应的坐标系是怎么样的？

答：对于 YJK 模型中导入的隔震支座局部坐标系，一般是方向 1，方向 2，方向 3 对应的是 X 轴，Y 轴，Z 轴；在后处理结果显示中，SF1、SE1 对应的是 X 轴的截面力与截面应变。

# 1.21 YJK 模型的减隔震构件导入后，有些参数出现了偏差，是什么原因？

答：由 YJK 导出\*.ydb 的阻尼器数据格式，会对部分参数进行四舍五入，自动去掉了小数点后两位，因此该数据导入 GFE 后会与 YJK 模型有些许偏差。

# 2.布尔运算相关问题

2.1 对土体进行布尔裁剪后，土层截面信息丢失了是什么原因？

答：如需裁剪后的土体（或其他图形）继承原有的截面属性，则在裁剪时需点选“替换原图形”。

2.2 对土体进行布尔裁剪后，形成了局部薄土层，是否会影响计算？

答：土体采用实体单元，当其布尔裁剪后形成薄土层时，网格剖分时，薄土层的位置可能会出现细长或体积较小的“畸形单元”，此类单元稳定步长较小，不利于计算。

在实际项目中，建议对钻孔的实际土层进行适当简化和合并，并且适当地调整地下室箱体裁剪位置的土层厚度，尽量避免薄土层的出现。

(可参见 3.1 条对分析模型合理性的判断)

2.3 对土体进行布尔裁剪后，网格划分失败，应该如何调整？

答：一般是因为 YJK 模型导入后自动生成得地下室外轮廓不规则，存在细小面或细小线，因此土体与该外轮廓布尔裁剪后会出现畸形面，无法划分正常网格。所以 YJK 模型导入后，应注意首先检查地下室外轮廓箱体是否正常。

# 3. 分析模型相关问题

# 3.1 分析模型完成后，如何初步判断模型的合理性？

# 答:

1. 首先可以从表观上对几何模型进行判断，查看各类构件是否规整、构件之间的连接关系是否正确，土体是否存在薄土层等情况；  
2. 可以从表观上对网格质量进行判断，查看是否存在畸形单元、网格划分异常等情况；  
(壳单元划分网格应尽量以 S4R 为主，尽量减少 S3R 单元，网格尺寸的建议取值可参看 3.2 条说明)  
3. 对分析模型进行模态分析，查看是否存在局部振动等异常情况；  
4. 对分析模型进行静力分析，查看是否存在局部位移异常等情况；  
5. 对分析模型进行短时间（可以取 1s）的动力分析，查看计算生成的\*.feasta 文件，搜索关键字“mass”，查看各种单元（B31、S3R、S4R、C3D4 等）的稳定步长及质量缩放情况；（如下截图所示）

![](../assets/GFE-FAQ/246237c0c9b156782c47b6a802bd3c865567bb0460b490bb7122b99d0bc023bd.jpg)

# 3.3 在 GFE 中的阻尼是如何输入的？

答：GFE 计算采用瑞利阻尼，可以通过输入阻尼比及频率，自动计算材料的瑞利阻尼，并自动写入到材料参数中。（如下截图所示）

2. 土体阻尼比根据实际情况确定（一般可取 0.05\~0.09），频率取“场地反应分析”中的土体第一模态频率。

# 3.4 在 GFE 中的进行地震动力计算时，是如何考虑重力荷载代表值的？

答：GFE 中以“附加质量+自重质量的形式”的形式考虑重力荷载代表值（质量）。

从 YJK 导入 GFE 时，程序会自动在“边界条件与荷载”中生成“Comb\_M\_xxx”附加质量，即为除自重外的重力荷载代表值转换的附加质量。一般为“(1.0 附加恒载+0.5 活载)/9.8”，其组合系数 0.5 可以在 YJK 中进行修改。

在进行地震动力计算时，地震工况中将“Comb\_M\_xxx”附加质量添加到 initial 分析步中，再加上程序自动考虑的自重质量，即为地震动力计算时总的重力荷载代表值（质量）。

# 3.5 在 GFE 中的进行模态分析时，应如何考虑附加质量？

答：从 YJK 导入 GFE 时，程序会自动在“边界条件与荷载”中生成“Comb\_M\_xxx”附加质量，即为除自重外的重力荷载代表值转换的附加质量。一般为“(1.0 附加恒载+0.5 活载)/9.8”，其组合系数 0.5 可以在 YJK 中进行修改。

在进行模态分析时，模态工况中将“Comb\_M\_xxx”附加质量添加到 initial 分析步中，再加上程序自动考虑的自重质量，即为模态分析的总质量。

该总质量与 YJK 周期计算时取的质量一致，但注意 YJK 中地震作用信息选“计算水平和反应谱方法竖向地震作用(整体求解)”，否则盈建科竖向质量会为 0，如下截图所示。

# 3.6 在 GFE 中写出 inp 文件时出现闪退，是什么原因？

答：一般是模型未进行网格划分导致的。

# 3.7 在 GFE 中计算中断/中止，是什么原因？

答：一般是以下几个原因导致：

1. 仅 GPU 计算中断，有可能是显卡配置不够或驱动版本太旧（GPU 计算要用 NVIDIA，建议用 6GB 以上，CUDA 为 11.0 及以上版本）；  
2. CPU、GPU 计算均中断，可能是电脑账户为中文名（需改成英文名），也可能是软件的“程序”文件夹的路径含有空格（需删除空格），也可能是“创建作业”的工作目录含有复杂的中文路径（建议采用英文路径）；  
3. 电脑硬盘的内存不足，无法生成结果文件；  
4. 电脑运行内存不足，模型太大，这种情况甚至会导致电脑关机重启；（一般 CPU 计算占用内存会比较多，GPU 计算占用内存会小一点）

5. 排查以上设置原因后，可能是模型存在问题：

a. 模型存在计算构件缺少截面属性；  
b. 模型存在重复的绑定约束；  
c. 模型存在多余的计算网格;  
d. 模型存在非常畸形单元导致发散；（可以使用 GFE 自带的移除畸形网格功能尝试计算）  
e. 写出 inp 文件时勾选了” 写出 vuel “;  
f. 目标时间增量设置太大，导致质量缩放太大；  
g. 人工边界误用了节点集等不恰当集合，应改为表面集；  
h. 非线性分析时材料的泊松比设置不合理。

# 3.8 在 GFE 中写出的 inp 文件，在软件 A 中无法计算是什么原因？

答:

1. 从 YJK 导入 GFE 的模型，程序会将 YJK 中的刚性杆转换为“两点弹簧 SPRING2”，而软件 A 的显式动力分析不支持 SPRING2 的计算；  
若需在软件 A 中计算，则需要在 GFE 中将“两点弹簧 SPRING2”改为“沿线弹簧 SPRINGA”；

2. 从 YJK 导入 GFE 的模型，程序会将桩与筏板分别导入，然后采用桩与筏板 embedded（嵌固）的方式进行连接，即线单元与面单元面外嵌固的方式连接，该嵌固方式在软件 A 中不支持；（桩与筏板分别导入，再采用嵌固方式连接的方式，可以很好地提高筏板的网格质量）

若需要在软件 A 中计算，则可以在 YJK 模型导入时根据项目实际情况填写“桩合并阈值”。使其值不为零，则桩与筏板的连接方式会变为“共节点”，但该做法可能会导致基础处出现小板及短线，影响网格划分。

3. 可能是畸形单元、质量缩放等原因导致。

4. 可能是 GFE 模型中场输出设置了 NFORC1、NFORC2，而软件 A 中不支持该种设置，将 inp

文件中的 NFORC1，NFORC2 改为 NFROC 即可。

# 3.9 在 GFE 中采用人工边界（黏弹性边界）计算地震作用时，写出的 inp 文件如何在 ABA 中计算？

答：在写出 inp 文件时，应勾选“高级”中的“写出 Vuel”，此时写出的 inp 中会额外多出一些人工边界单元，并且在 inp 文件路径下会多生成一个\*.vload 文件，在软件 A 中可通过子程序来处理这些单元的。（该子程序可在官网的产品下载页获取，也可以联系技术支持人员获取）

需要注意的是，此时的 inp 文件无法在 GFE 中计算，若需要在 GFE 中计算则写出 inp 文件时不可勾选” 写出 Vuel “。

# 3.10 在动力分析步中，质量缩放参数中的“目标时间增量”应如何合理地设置？

答:

1. 首先应确认网格划分合理，详见 3.2 条相关说明。  
2. 再按 3.1-4 条查看\*.feasta 文件，查看各类单元的稳定步长，“目标时间增量”为实际的动力迭代步长，其值应尽量小于等于各类单元的稳定步长。

当其值太大时会导致单元质量缩放（mass scaling）很大，使计算结果失真；但当其值太小时会导致计算时间很长。因此须设置一个正确值，使质量缩放合理（一般不大于 10%），并且计算时长合适。

# 3.11 在 GFE 中提交作业进行模态分析后，一直处于运行状态无法完成计算，是什么原因？

答：当电脑用户名为中文名或存在空格等非法字符时，可能会出现该问题，需修改电脑用户名为英文名等合法字符名称。

# 3.12 材料中含有塑性参数时，GFE 是否一定进行非线性计算？

答：在静力分析中，不管材料是否含塑性参数，计算时仅读取弹性参数，即最终计算结果为线弹性的计算结果；但在动力计算中，若材料含塑性参数，则计算会读取塑性参数，最终结果为非线性的计算结果。

# 3.13 弹性分析模型如何转换为非线性（弹塑性）分析模型？

答：在 GFE 中可以一键转换结构材料为线性材料/非线性材料，也可以一键生成配筋/移除配筋，其中生成配筋可以采用“读取 YJK 配筋“或”按默认生成配筋（默认参数可调整）”；关于结构的线性材料与非线性材料，也可以在 YJK 数据接口的界面对是否导入钢筋进行选择；

# 3.14 保存 GFE 模型或另存 GFE 模型会出现闪退，是什么原因？

# 3.16 网格划分失败，是什么原因？

答：主体结构网格划分失败一般是由于不共面墙、或存在微小的畸形面导致；土体网格划分失败，一般是由于结构的地下室外轮廓存在微小面，导致土体布尔裁剪出现问题，或者土体多次裁剪出现微小面导致出现问题；一般可以修改 YJK 模型进行解决。

# 3.19 桩与土嵌固的模型，模态分析结果为什么异常？

答：一般是因为桩和底板没有共节点也没有嵌入，桩仅嵌入了土中，而“线”与“体”的嵌固是无法约束转动自由度的，因此会导致桩模态分析时转角很大，结果异常。

# 3.20 场地反应分析的结果异常，一般是什么原因？

答：一般是因为地震波输入的位置不太正确：

1. 基岩波要在基岩处输入，地表波要在地表处输入，基岩露头波要在基岩露头处输入；  
2. 每个位置的波形都不太一样，不能将地表波在基岩处输入，否则场地反应分析的结果是不合理的；因为基岩处的地震动，是上行跟下行两种波形的叠加，不是随便给个地震动，都可以得到合理的结果的；  
3. 比如阪神波，应该和 peer 上面的实测波一样，是属于地表波，应该在地表输入比较合理。

# 3.21 场地反应分析中的高级参数是否需要调整，一般怎么调整比较好？频域法与时域法的结果应该如何选择？

答：一般只需要调整两个参数即可” UseIntgOutp “及” UseEERAMat “，下面是参数说明：

1. UseIntgOutp=true，采用时域法分析结果；UseIntgOutp=false，采用频域法分析结果。  
2. UseEERAMat=true，采用等效线性化，需有试验数据曲线时才生效；UseEERAMat=false，不采用等效线性化。

![](../assets/GFE-FAQ/8747f0893f5f5e180e05d869b030b556305d080387b10d1acdfa7e63de69fead.jpg)

针对是否具有土层试验数据曲线的情况，一般建议按下述原则调整参数：

# 情况一：土体有试验数据曲线：

1. 小震/中震弹性分析：

a）场地反应分析：useEERAmat=true，等效线性化；  
b）软件自动替换 EERA 土体材料、自动输入 Alpha 阻尼；（inp 中可以看到）

c) 地震作用采用场地反应分析的“时域法”结果；UseIntgOutp=true;

2. 大震弹塑性分析:

a）场地反应分析：useEERAmat=false;  
b）手动操作，一键将试验数据曲线算出 A、B、γ0 并填入土体材料；（无需另外输入阻尼）  
c) 地震作用采用场地反应分析的“频域法”结果；UseIntgOutp=false;

# 情况二：土体无试验数据曲线：

1. 小震/中震弹性分析：

a）场地反应分析：useEERAmat 功能不生效；  
b）手动操作，根据土体阻尼比(一般 5%\~9%)与场地反应分析得出的一阶土体模态，一键算出 Alpha 阻尼并填入土体材料中；  
c) 地震作用采用场地反应分析的“时域法”结果；UseIntgOutp=true;

2. 大震弹塑性分析:

a）场地反应分析：useEERAmat 功能不生效；  
b) 手动操作，将摩尔库伦参数填入土体材料中；  
c) 手动操作, 根据土体阻尼比(一般 7%)与场地反应分析得出的一阶土体模态, 一键算出 Alpha 阻尼并填入土体材料中;  
d）地震作用采用场地反应分析的“时域法”结果；UseIntgOutp=true。

# 3.22 场地反应分析的结果，感觉设置好后又发生了变化，甚至出现了前处理建模结果与计算完成结果不一致的情况，一般是什么原因？

答：场地反应分析的结果与土体材料的阻尼有关，一般首次运行场地反应分析后可以得到土体一阶模态及最大剪应变等信息，然后会利用这些信息去生成土体材料的瑞丽阻尼并自动输入，这一步结束后应再进行场地反应分析，此时的得到的结果才是正确的最终结果。

# 3.23 土-结整体模型中，土压力是怎么考虑的？

答：在土-结整体模型中，”土体密度+重力加速度“会对土体产生竖向的压缩变形，竖向变形在土体”泊松比“的影响下会产生横向变形，从而对结构产生侧向挤压的推力。

# 3.24 Connector 连接器的参数，如果不勾选 F2、F3 的话，是默认 2、3 两个方向的没有刚度还是 2、3 两个方向不放开自由度呢？

# 3.25 GFE 中搜索土-结接触、自动创建绑定约束时，自动生成的表面集异常，一般是什么原因？

答：一般是两种原因：原因一是结构与土体存在微小重叠，即自动生成的结构地下室外轮廓与结构地下室的最外围结构没有完全重合，因二是土体未划分与结构接触面相匹配的表面。

答：一般是因为钢筋太零散了，太分散的钢筋进行嵌入时可能会识别异常，建议将钢筋尽量合并在一个几何体中，再进行嵌入。

# 4. 计算结果相关问题

4.1 人工边界（黏弹性边界）计算地震作用时，用批处理命令提交计算，计算结果缺失人工边界、地震场地反应分析等信息是什么原因？

答：使用批处理命令时，inp 文件路径中缺少了 inpx 文件，该文件包含“构件偏心、地震载荷、人工边界、工况、空间分布函数、包络输出”等信息。

4.2 纯结构模型（无土模型，结构底部基础全约束）动力计算不发散，但是土-结模型动力计算会发散，一般是什么原因？

答：纯结构模型底部基础全约束，若基础存在畸形网格或其他问题时，不会出现发散情况；而土-结模型的结构基础并非全约束，因此会暴露其问题而出现计算发散。

答: 可以将动力分析的总时长改成较小值 (如 0.02s)，场输出整个模型，频率改为更小值 (如 0.001s)，然后打开计算结果 db 文件，逐帧查看是第几帧开始出现问题，在该帧查询“场”，找到最大位移节点。（如下截图所示）

# 4.4 导出计算书出现闪退，是什么原因？

答: 可能是 Micro office 或 WPS 版本的兼容性问题, 可以尝试安装不同版本进行尝试。也可能是 Office 未激活或未正确安装。

# 4.5 输入的爆炸荷载很大, 但计算结果显示的爆炸荷载响应 IWCON 数值偏小, 是什么原因?

答：设置的场输出时间间隔太大了，爆炸荷载加载速度快，衰减也快，应将场输出的频率尽可能设为较小值，输出的结果会更准确。

(比如场输出间隔 0.02s，第一帧 0.02s 没有出现响应，0.04s 才有响应，那 0.04s 这个响应不一定是最大值，最大值可能出现在 0.03s，只是该帧结果没输出。)

# 4.9 在 “轴压比云图/DWG 图” 中调整荷载组合的地震组合系数，最终计算结果无变化，是什么原因？

答：一般是两种原因，一种是上述 4.8 条的原因，没有读取到动力时程分析的计算结果；

还有一种可能是地震工况相对与其他工况产生的内力很小, 导致轴压比云图变化不明显; 且 DWG 图为所有组合工况的包络输出, 只输出最大内力及配筋结果, 地震组合参与的工况可能不是控制工况。

# 4.10 GFE 中输出结果的单位是什么？

答：在 GFE 中没有指定的单位制，内部的实现是无量纲。在工程项目中常用的一套单位制度是 m、kN、t、kPa、t/m³、m/s²。如果 GFE 模型是从 YJK/PKPM 模型导入的，则按此单位制执行即可。

# 4.11 GFE 非线性分析结果异常的原因是什么？

答:

1. 边界条件设置错误，初始分析步（Initial）中不应有土体支座边界条件；应在静力分析步施加土体底部固定、土体四周法向约束的边界条件；应在动力分析步施加人工边界；  
2. 重力加载对象错误，重力加载的作用对象应为“整个模型”，不应只加载到结构上；  
3. 静力分析步的场输出设置错误，由于非线性分析需考虑初始应力条件；在非线性分析时，动力分析接静力分析软件会自动进行地应力平衡；但是静力分析步的场输出中必须输出 U、UR、RF、S，才可以在动力分析步中读取到相关的数据。

# 4.12 GFE 非线性分析时，需要进行动力接静力计算，在动力分析步的前几秒地震波仍较小时，为什么会出现较大的水平向位移、加速度震荡？

答：GFE 动力接静力自动执行地应力平衡，然后再在动力步中加载重力，该重力为瞬时加载，而整个有限元分析体系的本质即为“质量-弹簧-阻尼”体系，因此瞬时加载必然会导致震荡；如果想尽量减少或消除这种震荡效应，可以将重力沿较长时间缓慢加载，或者施加非常大的阻尼、弹簧刚度。而且显式动力分析接隐式静力分析时，存在显式单元与隐式单元的转变，单元差异也会导致一定的震荡。

# 4.13 从计算结果看，绑定约束 Tie 出现了结果异常，是什么原因？

答：可能是主从面的关系不太正确。一般按如下原则执行：

1. 对于三维模型来说，绑定约束一般是土做主面，结构做从面，即粗网格做主面。  
2. 对于二维平面应变模型来说，一般是结构做主面，土做从面，即细网格做主面。

# 4.14 从计算结果看，地下室结构柱的轴力/轴压比很大，或者混凝土损伤非常严重，一般是什么原因？

答：可能是结构顶板重复考虑了覆土荷载导致的，在 GFE 中顶板的覆土重量是通过“土密度+重力加速度”形成的，无需再在地下室顶板输入附加质量或附加压力，否则会导致重复考虑附图荷载，导致地下室结构柱轴力/轴压比很大，或者大面子柱混凝土损伤非常严重。

# 4.15 模态分析时，YJK 振型与 GFE 振型相差远的原因，一般是什么原因？

答：常见原因如下：

1. YJK 模型计算模态时不包含基础，GFE 一般为整体模型进行计算分析。  
2. YJK 模型中有刚性楼板假定、中（边）梁刚度调整系数、连梁刚度折减系数等参数，会对结构固有频率影响较大。  
3. 在 YJK 中对连梁模拟一般有两种方式：梁元或壳元。在导入 GFE 时，连梁会根据 YJK 数据接口的导入参数，生成为壳元或梁元。两个模型的连梁可能存在差异。  
4. 钢筋影响：YJK 中进行模态分析时不考虑钢筋影响，仅考虑混凝土部分的刚度。而 GFE 可以考虑钢筋的影响，也可以不考虑。  
5. YJK 模型导入 GFE 后, 需输入网格控制参数重新划分网格, 而 YJK 模型中的网格是默认生成的, 两个模型的网格可能差异较大。

# 5.3 GFE 软件使用时建议电脑配置是怎么样的？

答：软件名称：GFE 2024

并行核数需求：

CPU：无硬性要求，建议 intel cpu，12 线程以上

GPU: nvidia 显卡，显存 8G 以上，支持多卡并行。建议用专门的计算卡，例如 V100。多卡并行的话，卡与卡之间的互联建议用 nvlink

命令行计算命令：gfe-daemon-dat INP 路径 -gfedir 结果目录

# 5.4 GFE 模型文件，从作业管理器查看计算结果时出现闪退，一般是什么原因？

答：提交作业后如果未计算完成，未提示 success 就点查看结果，会导致模型闪退。

# 5.5 GFE 前处理模块，过滤器无法正常使用，一般是什么原因？

答：功能是正常的，但由于几何体实际上既有角点又有表面和内部实体，因此如果仅选择隐藏实体，可能还会有表面存在。导致用户误以为过滤器无法使用，但实际上是合理的。

# 5.6 GFE 模型无法保存，只能另存，一般是什么原因？

答：可能因为误操作或系统原因，导致模型属性变成了只读，可以右键模型文件，打开属性选项卡并取消“只读”。

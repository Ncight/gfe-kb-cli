# GFE 对象引用关系总览

> 梳理 GFE 前处理中各功能模块的**正确引用关系**：谁引用谁、通过什么类型的集合/表面来关联。  
> 依据：GFE 实际案例操作手册 v2025 + 用户手册 v2025 + 命令流手册 v2026.03

---

## 核心引用关系图

```
几何体 (Geometry)
  ├─→ 几何集 (gset) ──→ 截面属性 (Section)    [通过集合名引用]
  │                  ──→ 边界条件 (BC)         [约束作用区域 = 集合/节点集]
  │                  ──→ 嵌入区域 (Embed)      [嵌入区域+主区域 均为集合]
  │                  ──→ 体力/惯性力荷载        [作用区域 = 几何集/单元集]
  │                  ──→ Model Change 生死     [elemAdd/elemDel = 几何集名]
  │
  ├─→ 表面集 (Surface) ──→ 绑定约束 (Tie)      [主从表面对]
  │                     ──→ 压力荷载            [作用区域 = 表面]
  │                     ──→ 人工边界            [边界面 = 表面集(施加时转单元集) + 形心参数]
  │                     ──→ 面面接触            [表面对 contact.surface=[主,从]]
  │
  ├─→ 材料 (Material) ──→ 截面属性 (Section)   [通过材料名引用]
  │
  └─→ 网格 (Mesh) ──→ 节点集/单元集 (自动生成)
```

---

## 一、截面属性 → 几何集（集合）

**规则**：截面属性通过**几何集**（集合）来指定作用区域，同时引用材料名。

| 要素 | 引用对象 | 引用方式 |
|:-----|:---------|:---------|
| 截面名 | — | 自定义命名 |
| 材料 | 材料对象 | `mat_name` 引用材料名称 |
| 作用区域 | **几何集** | `elset_name` 引用集合名（gset 名可直接作 elset_name 用，§8.2 实践；"自动生成同名单元集"机制待实测） |

**实际案例操作步骤**：
1. 先创建几何集（选中一个或多个几何体），如 `Set-土层1`
2. 创建截面属性时，"集合"字段选择对应几何集
3. 截面类型（实体/壳/梁）决定材料-几何的绑定方式

```python
from GFE.Pre import section, set as setmod
from GFE.Pre.geometry import geo_mgr

# 先为几何体创建几何集
g = geo_mgr().find('土层1')
setmod.gset_mgr().add('Set-土层1', [g.shape()])

# 截面属性引用几何集
s = section.property_solid()
s.name = 'Sect-土层1'
s.elset_name = 'Set-土层1'  # 引用几何集名称
s.mat_name = 'Mat-ZTT'      # 引用材料名称
section.sect_mgr().add(s)
```

> **注意**：手册案例中，"这里创建集合是为了方便后续赋予截面属性"。
> 若几何体仅有一个，案例中可直接用几何名作为集合名（机制疑为建模/YJK 导入时自动建同名集，待实测；保险写法是始终显式 `gset_mgr().add`）。

## 二、边界条件 → 集合（几何集/节点集）

**规则**：位移约束（全约束、滚轴、固定）等边界条件通过**集合（面构成的几何集 / 节点集）**来指定作用区域；**表面集不用于 BC**——表面集用于 Tie/压力/人工边界。

| 边界类型 | 作用区域类型 | 典型用法 |
|:---------|:-------------|:---------|
| 全约束 (固定) | 几何集（面构成）/ 节点集 | 底面全约束 U1=U2=U3=0 |
| 位移/转动约束 | 几何集（面构成）/ 节点集 | 侧面法向约束 |
| 压力载荷 | **表面集**（必须） | 面压力 |
| 集中力/力矩 | 节点集 / 节点ID | 桩顶荷载 |
| 惯性力/体力 | 几何集 / 单元集 / 整个模型 | 自重 g |

**实际案例操作步骤**（底面全约束）：
1. 先创建由底面 face 构成的**几何集**（"建立相应的集合"）
2. 创建边界条件时，"区域"选择该集合
3. 设置约束类型和自由度

```python
from GFE.Pre import set as setmod, boundary

# 创建底面几何集（由底面 face 构成，非表面集）
setmod.gset_mgr().add('bot', [face_bottom, ...])

# 边界条件引用集合
bc = boundary.boundary()
bc.name = 'BC-Bottom'
bc.type = 0         # 0=完全约束(CONSTRAIN)；位移约束是 type=1（§2.10.3）
bc.set = 'bot'      # 引用集合名（几何集/节点集），非表面集
bc.valid_dof = 0    # ⚠ 位掩码 1=开放 0=限制（与 Abaqus 相反）：全约束=0；7 反而是放开三平动
bc.value = [0]*6
boundary.bc_mgr().add(bc)
```

> **案例参考**：手册 §12.13 "底面全约束"——先建立底面集合，再引用。
> "底部的全约束。在进行该步骤前，需要建立相应的集合，分别是由土层底面构成的几何、土体y向的外表面、土体x向的外表面。"（该引文本身即证明 BC 引用**集合**）

## 三、绑定约束 (Tie) → 表面集

**规则**：绑定约束通过**表面集对**（主表面 + 从表面）来定义。

| 要素 | 引用对象 | 说明 |
|:-----|:---------|:-----|
| 主表面 (first_surf) | **表面集** | GFE 案例惯例：土/大域作主面（手册未给一般规则） |
| 从表面 (second_surf) | **表面集** | 案例惯例：结构作从面 |
| 搜索容差 | 数值 | 判定面对是否匹配 |

**实际案例操作步骤**：
1. 方法A（推荐）：工具栏 → 通用 → **搜索接触**
   - 设置查找区域和搜索容差
   - 点击"搜索"自动发现接触对
   - 选择"绑定"类型 → 确定
   - **GFE 自动创建表面集和绑定约束**
2. 方法B（手动）：手动创建两个表面集 → 树状列表 → 相互作用 → 创建绑定约束

```python
from GFE.Pre import surface, interaction

# 方法A：搜索接触（推荐——GUI 版全自动；API 版只返回面对，
# 需自行据 pairs 建 surface + surface_pair，见命令流手册 §8 完整遍历段）
from GFE.geometry import contact_pair as cp
pairs = cp.search_face(shape1, shape2, tolerance=1.0)

# 方法B：手动创建
ms = surface.geometry_surface('Surf-Master')
ms.data = [[geo_id1, face_id, 0], ...]
surface.surf_mgr().add(ms)

ss = surface.geometry_surface('Surf-Slave')
ss.data = [[geo_id2, face_id, 0], ...]
surface.surf_mgr().add(ss)

sp = interaction.surface_pair()
sp.name = 'Tie-1'
sp.first_surf = 'Surf-Master'   # 主表面集
sp.second_surf = 'Surf-Slave'   # 从表面集
interaction.tie_mgr().add(sp)
```

> **案例参考**：手册多章节均采用 "搜索接触 → 绑定约束" 工作流，GFE 自动生成表面集。

## 四、接触 (Contact) → 表面对

**规则**：面面接触通过**表面对**定义（`contact.surface = [主表面, 从表面]`，命令流 §2.13.3）。

| 要素 | 引用对象 | 说明 |
|:-----|:---------|:-----|
| 接触区域 | **表面对** | `surface=[主,从]`，两侧均为表面集 |
| 接触属性 | 摩擦系数等 | 法向/切向行为 |

> **注意**：搜索接触工具也可自动创建接触对（自动生成表面集）。

## 五、嵌入区域 (Embed) → 集合 + 集合

**规则**：嵌入约束的**嵌入区域和主区域均引用集合**（几何集/单元集）；INP 层 *Embedded Element 两端皆 elset。

| 要素 | 引用对象 | 说明 |
|:-----|:---------|:-----|
| 嵌入区域 | **集合**（几何集/单元集） | 被嵌入的结构（如钢筋）；属性是**列表** |
| 主区域 | **集合**（几何集/单元集） | 宿主实体（如混凝土）；属性是单个字符串 |

```python
from GFE.Pre import interaction

# 嵌入约束：钢筋嵌入混凝土（属性实名 embedded_names / host_name，spec L385/387）
embed = interaction.embed()
embed.embedded_names = ['Set-Rebar']   # 列表
embed.host_name = 'Set-Concrete'       # 字符串
interaction.embed_manager().add(embed)
```

> **案例参考**：手册 §10.9——"(7) 设置嵌入区域，设置嵌入区域以及主区域"（§2.2 钢骨混凝土柱同为嵌入用例）。
> 嵌入有双容差（绝对/比例）且两真值源数值有差异，照抄前核对（merged #94）。

## 六、人工边界 → 表面集

**规则**：人工边界（如黏弹性边界）通过**表面集**来指定边界面。

| 要素 | 引用对象 | 说明 |
|:-----|:---------|:-----|
| 边界面 | **表面集** | 土体外边界面（GFE 施加时内部转单元集） |
| 结构形心 | 几何 / 坐标 | 一键施加还需指定形心参数（案例 §10.10；inpx 缺省静默用 (0,0,0)） |

**实际案例操作步骤**：
1. 先创建土体边界的**表面集**
2. 通过人工边界交互界面，选择表面集
3. 设置边界类型和参数

> **案例参考**："首先需要建立土体边界的表面集，再通过人工边界交互界面进行人工边界的建立。"

## 七、荷载类型 → 引用方式速查

| 荷载类型 | 作用区域 | Python `bc.set` 引用 |
|:---------|:---------|:--------------------|
| 位移/全约束 | 表面集 | 表面集名称 |
| 集中力/力矩 | 节点集 / 节点ID | 节点集名称 |
| 压力 | **表面集** | 表面集名称 |
| 体力 | 几何集 / 单元集 / 整个模型 | 几何集名称或空 |
| 惯性力 | 整个模型（默认） | 空字符串 |
| 线荷载 | 单元集 / 单元ID | 单元集名称 |
| 温度 | —（bc type=14 手册标"未使用"；温度初值走 InitTemperature 离散场） | — |
| 孔压 | 节点集 / 几何集 | 集合名称 |
| 水压力 | 表面集 | 表面集名称 |

## 八、工况 (Case) → 引用方式

| 要素 | 引用对象 | 说明 |
|:-----|:---------|:-----|
| 分析步列表 | 分析步名称 | `c.steps = ['Initial', 'Geostatic-1', ...]`，必须显式激活 |
| 边界条件 | BC 名称列表 | `set_bcs(step, [bc_names])` |
| 钝化（elemDel） | **几何集名称** | `set_elemDel(step, [gset_names])` |
| 激活（elemAdd） | **几何集名称** | `set_elemAdd(step, [gset_names])` |
| 场输出请求 | 输出请求名称 | `set_fieldReqs(step, [fo_names])` |
| 历史输出请求 | 输出请求名称 | `set_histReqs(step, [ho_names])` |
| 振动荷载（SSI 必用） | 荷载名称 | `set_vload(step, [vl_names])` |
| 人工边界（SSI 必用） | 人工边界名称 | `set_artbc(step, [ab_names])` |
| 初始条件 | 初始条件名称 | `set_initialConditions(step, [ic_names])`（静→动接力挂载相关） |

> Model Change（生死单元）中的钝化/激活引用**几何集名称**（GSet-*）。

### 施工步生死单元对应关系（铁塔荷载对隧道影响工况）

> 来源：本项目建模 session 记录（非手册案例），数值自洽但无手册真值可核。

| 分析步 | 物理意义 | 钝化 (elemDel) | 激活 (elemAdd) | 存活单元 |
|:-------|:---------|:---------------|:---------------|:---------|
| Geostatic-1 | 天然地层初始应力 | GSet-桩, GSet-隧道衬砌1, GSet-隧道衬砌2 | **无**（土层从初始状态即激活，无需 elemAdd） | 三层土+两隧道土 |
| Static-1 | 隧道开挖+衬砌安装 | GSet-隧道土1, GSet-隧道土2 | GSet-隧道衬砌1, GSet-隧道衬砌2 | 三层土+两衬砌 |
| Static-2 | 位移参考基准 | — | — | （同上） |
| Static-3 | 桩体激活 (wished-in-place) | — | GSet-桩 | 三层土+两衬砌+桩 |
| Static-4 | 施加铁塔荷载 | — | — | （同上，加集中力） |

**施工逻辑**：
- Geostatic-1 建立天然场地的初始地应力，此时桩和衬砌尚未施工（钝化）。**注意：土层单元从初始状态即激活，不可再 elemAdd，否则报"重复添加"错误**（实测经验，手册无载）
- Static-1 模拟隧道开挖（删除隧道土体）并安装衬砌（激活衬砌壳单元）
- Static-2 重置位移场，作为后续增量位移的参考基准
- Static-3 桩体 wished-in-place（直接激活，不模拟打桩过程）
- Static-4 在桩顶施加 10 吨（≈100 kN）铁塔集中力荷载

## 九、完整操作流程（以岩土模型为例）

```
1. 建立几何体 → 命名（土层1/桩/隧道衬砌1 等）

2. 创建几何集（集合）
   - 为每个几何体创建同名几何集（用于截面属性）
   - 或直接使用几何名（GFE 自动关联）

3. 创建表面集（表面）
   - 底面表面集 → 用于底面全约束
   - 四侧面表面集 → 用于法向约束
   - 接触面表面集 → 用于Tie/接触（或用搜索接触自动生成）

4. 定义材料 → 指定密度/弹性模量/本构

5. 定义截面属性 → 引用 几何集 + 材料名

6. 建立绑定约束 → 通过搜索接触自动生成（引用表面集对）

7. 建立边界条件 → 引用 表面集
   - 底面全约束 → Surf-Bottom
   - 侧面法向约束 → Surf-XMin/XMax/YMin/YMax

8. 建立荷载 → 引用 几何集/表面集/节点集（按类型）

9. 定义分析步 → 设定步类型和增量参数

10. 配置工况 → 引用分析步 + BC + Model Change（几何集名）

11. 输出请求 → 绑定到工况每步

12. 网格划分 → 对几何体划分

13. 导出 INP → 指定工况导出
```

## 十、常见错误与纠正

| 错误做法 | 正确做法 | 原因 |
|:---------|:---------|:-----|
| 截面属性直接用几何名 | 截面引用几何集（集合） | 截面通过集合关联 |
| 边界条件误用表面集 | 边界条件引用**集合（几何集/节点集）** | BC 走集合；表面集用于 Tie/压力/人工边界 |
| 手动创建Tie的表面集 | 用**搜索接触**自动生成 | 效率高且不易遗漏 |
| Tie引用几何集 | Tie引用**表面集对** | 绑定需要面的正反属性 |
| Model Change引用几何体名 | Model Change引用**几何集名**(GSet-*) | set_elemAdd/set_elemDel 实传几何集名（Cmd §8.2 'kaiwaN'） |

> 用 FEM-SPH 流固耦合（fluid-structure interaction, FSI）模拟一个盛水圆柱水桶在底部 X 向加速度激励下的晃动：壳单元水桶用有限元算变形受力，桶内水用 SPH（光滑粒子流体动力学 smoothed particle hydrodynamics）粒子算流体晃荡。练的是 GFE 的「流体动力」模块、SPH 对象定义、流体动力分析步、以及 FEM 预计算 + 批处理提交 SPH 求解器的两段式工作流。

## 工程背景与模型
- 几何：圆柱体水桶半径 0.5 m、高 1.5 m、壁厚 0.005 m；桶内水半径 0.5 m、高 1.0 m。
- 水桶建模：草图画半径 0.5 m 圆 → 拉伸 (0,0,1.5) 出侧面 → 填充区域得底面圆 → 布尔合并（Merge-1）成桶；水体由底面圆拉伸 (0,0,1.0) 得到（Exturde-2）。
- 工况：水桶底面除 X 向外全部位移约束（U2=U3=UR1=UR2=UR3=0），底面沿 X 向施加峰值 1.0 m/s² 的加速度时程载荷（幅值函数取预设地震波 `25_RH1TG025_(RenGong_T_025)_x_Zhu`）。
- 水桶材料（钢，Material-Tong）：密度 7800 kg/m³，杨氏模量 2.06e11 Pa（即 206 GPa），泊松比 0.25，瑞利阻尼 α=0、β=0。
- 单位制：SPH 流体动力分析强制使用 SI(m) 国际标准单位 —— kg、m、s、N、Pa、kg/m³、m/s。材料参数必须按此单位输入，否则结果错误。

## 演示的 GFE 功能
- **FEM-SPH 流固耦合分析**：有限元（壳水桶）与 SPH 粒子（水）耦合，是本案例核心，区别于 [[GFE-Cases-08-SPH小球落水]]（同册 SPH 案例一）。
- **壳单元建模**：水桶用各向同性壳截面（Property-tong），壳厚 0.005 m，积分点 5 个；全局网格参数中单元形状选「四边形/六面体」、2D 剖分算法切换为 Frontal-Delaunay for Quads，四边形单元 S4R 主导（允许少数 S3 三角形，图 9.3-5）。
- **流体动力 / SPH 对象设置**：模型树「流体动力」节点下，把水体几何（Exturde-2）定义为「流体」对象（填充模式=全部），把水桶几何（Merge-1）定义为「固体」对象并勾选「设置为浮体」（浮体密度 7800 kg/m³）。
- **流体动力分析步**：程序选「线性摄动 → 流体动力」，含「定义常量 / 对象顺序 / 求解参数」多页签设置。
- **FEM 场输出 + SPH vtk 双结果**：FEM 部分输出位移/转动位移/速度（节点）与应力/应变（单元）；SPH 部分输出流体粒子 vtk。
- **加速度载荷 + 幅值函数（时程）**：底部加速度激励，幅值函数取 GFE 预设人工地震波（名称含 RenGong=人工）；本案例非土-结构地震分析，仅借时程做激励。

## 关键操作与参数
- **网格尺寸要与粒子间距匹配**：水桶网格尺寸取 0.1 m，一般取 SPH 内部粒子间距的 5~10 倍（手册建议值）；硬约束是网格尺寸不得小于粒子间距。坑：网格若过小（甚至小于粒子间距），单元体内无法生成粒子，后续计算直接报错。
- **SPH 分析步求解参数**：内部粒子距离 0.01 m，勾选「自动计算 SPH 计算区域」；分析时长 1.5 s 与输出数据间隔 0.01 s 为本案例设置项（场输出时间间隔必须与此一致），其余保持默认——初始分析步时间 0.0001 s（默认勾选「自动计算」）、最小分析步时间 1e-5 s、最小/最大有效密度 700 / 1300 kg/m³、超出区域粒子比例阈值 1% 均为对话框默认值（src 行 80129）。
- **「对象顺序」页签**：需依次登记 SPH-water 与 SPH-tong 两个 SPH 对象。
- **创建工况（§9.6，作业前置）**：创建 SPH 工况 Case-WaterBucket——将 SPH-1 流体动力分析步加入激活组分析步；BC-base 边界条件分配到 Initial 初始分析步；加速度载荷 Acc-X 与场输出请求 FieldOutput-1 分配到 SPH-1 分析步（手册 L3230-3232），再创建作业。
- **两段式提交（FSI 必须）**：
  1. 作业管理器中创建作业，高级设置勾选「写出 SPH xml 文件」；
  2. 先对有限元部分（水桶）做**预计算**，生成 `.fea` 与 `.db` 文件供 SPH 求解器读取 —— 预计算不必算完，算到 1% 即可中止；
  3. 「写出 inp 文件」后工作目录生成 inp、`_Def.xml`（SPH 求解文件）、`mk2inpmap.txt`；
  4. 批处理脚本中改作业名、设 GFE 数据路径（gfedatapath=预计算结果文件夹）、GFE 文件名（gfefilename=该文件夹内 .fea 文件名）、SPH 程序路径，双击 .bat 提交 SPH 计算。
- **坑：mk2inpmap.txt 必须就位**。该文件记录 inp 单元号 ↔ xml 浮体 mk 值的映射，是 FSI 的必要文件；批处理提交 xml 前必须把它放进有限元预计算结果文件夹，否则耦合失败。
- **后处理双视图**：打开 `.db` 看有限元结果（水桶位移/应力云图）；切到「流体动力」页签打开 vtk 看流体粒子 —— vtk 在「作业名_out / particle」文件夹内，选后缀 `_Fluid` 的流体粒子 vtk 文件。

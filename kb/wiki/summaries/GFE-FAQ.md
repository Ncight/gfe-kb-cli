## 关键章节

### 1. YJK 模型导入
- YDB 必须文件：`dtlmodel.ydb`（模型荷载）+ `dtlCalc.ydb`（设计结果），均在 YJK 计算路径的"施工图"子文件夹；可选 `Jccad_0.ydb`（基础）。
- 检查顺序（1.17）：先看地下室外轮廓箱体是否正常 → 筏板是否正确（多余/漏洞）→ 不生成基础导入后对比质量与 YJK 是否一致。
- 常见建模错误（1.16）：上下层墙体须共轴线居中对齐（外墙偏轴距离统一改 0）；相对楼层标高建模易出错，宜改 0 或拆层；梁端/柱节点与墙线距离 <300 会让几何运算失败，须挪到墙线；不可出现环形（复连通）楼板。
- 隔震支座导入会转成 Connector 连接器；楼层组装错误会让多类支座合并成一个（1.18）。默认读非线性刚度，做模态/静力须手动改"等效刚度"（1.19）。局部坐标方向 1/2/3 对应 X/Y/Z，SF1/SE1 为 X 向截面力/应变（1.20）。

### 2. 布尔运算
- 裁剪要继承原截面属性须点选"替换原图形"（2.1）。
- 薄土层 → 细长/小体积"畸形单元"，稳定步长小不利计算；建议简化合并钻孔土层、调整箱体裁剪位置厚度（2.2）。

### 3. 分析模型（核心技术章）
- 模型合理性自检（3.1）：表观→网格质量→模态→静力→短时（1s）动力。短动力跑完查 `*.feasta` 搜 "mass" 看各单元（B31/S3R/S4R/C3D4 等）稳定步长与质量缩放。
- 网格取值（3.2）：钢混结构 0.8–1.5 m 并勾"合并全部"(Recombine2d)；土体不得大于"一维土层计算"的建议尺寸。剖分算法默认 Frontal-Delaunay for Quads。
- 阻尼（3.3）：GFE 用[[瑞利阻尼]]，输阻尼比+频率自动算并写入材料。结构频率取纯结构第一模态（地铁车站等全埋结构取土体第一模态），土体阻尼比一般 0.05–0.09、频率取场地反应的土体第一模态。
- 重力荷载代表值（3.4/3.5）：以"附加质量+自重"形式考虑，导入自动生成 `Comb_M_xxx`（一般 `(1.0 附加恒载+0.5 活载)/9.8`），加到 Initial 步。模态分析要与 YJK 周期质量一致，须在 YJK 选"计算水平和反应谱方法竖向地震作用（整体求解）"否则竖向质量为 0。

### 4. 计算结果排查
- 发散定位（4.3）：把动力总时长改极小（0.02 s）、场输出整模型频率改小（0.001 s），逐帧看哪帧开始异常并查最大位移节点。
- 单位制（4.10）：GFE 内部无量纲；YJK/PKPM 导入按 m、kN、t、kPa、t/m³、m/s²。
- 非线性结果异常（4.11）：Initial 步不应有土体支座边界；静力步施加土底固定+四周法向约束、动力步施加人工边界；重力须加载到"整个模型"；静力步场输出必须含 U、UR、RF、S 才能被动力步读取。
- 地下室柱轴力/损伤过大（4.14）：顶板覆土重量已由"土密度+重力加速度"形成，切勿再在顶板加附加质量/压力，否则重复计入。

### 5. 软件操作
- 推荐配置（5.3）：Intel CPU 12 线程以上；NVIDIA 显存 §5.3 推荐 ≥8GB（建议计算卡如 V100，多卡用 NVLink，无 CUDA 要求），§3.7 另一处写 6GB+、CUDA≥11.0——两处口径冲突待仲裁。命令行计算：`gfe -daemon -dat <INP路径> -gfedir <结果目录>`（三个独立参数 token，-dat 接 INP 而非 .pre）。

## 核心用法与参数
- **瑞利阻尼**（3.3）：结构阻尼比按实际取，频率=纯结构第一模态频率（全埋结构取土体第一模态）；土体阻尼比 0.05–0.09，频率=场地反应土体第一模态频率。
- **质量缩放目标时间增量**（3.10）：先确认网格合理 → 查 `*.feasta` 各单元稳定步长 →"目标时间增量"取 ≤ 稳定步长。太大→质量缩放过大（须 ≤10%）结果失真；太小→计算超长。稳定步长合理区间约 1e-4 ~ 1e-7（量纲秒），出现 1e-8 级多为畸形单元。
- **重力荷载代表值**：`Comb_M_xxx = (1.0 附加恒载 + 0.5 活载)/9.8`，组合系数 0.5 在 YJK 可改。
- **弹性转弹塑性**（3.13）：一键转换线性/非线性材料 + 一键生成/移除配筋（读 YJK 配筋 或 按默认）。默认配筋率示例：梁 0.01、柱 X/Y 各 0.0025、墙 0.004、构造柱 0.01、保护层 0.025 m。注意：改过构件截面名称会让钢筋转换闪退（名字索引断裂），不建议改名。
- **塑性是否生效**（3.12）：静力分析只读弹性参数（结果线弹性）；动力分析才读塑性参数（结果非线性）。
- **场地反应分析高级参数**（3.21）：`UseIntgOutp=true`→时域法，`false`→频域法；`UseEERAMat=true`→等效线性化（需试验曲线）。有试验曲线时小/中震用等效线性化+时域，大震用 A/B/γ0 填料+频域；无试验曲线则用软件一键计算的 Alpha 阻尼 + 时域法（§3.21 情况二）；大震另补摩尔库伦参数。运行后须用得到的土体一阶模态/最大剪应变再跑一次场地反应，第二次结果才正确（3.22）。
- **Connector 方向**（3.24）：不勾 F2/F3 = 该方向无刚度（不是约束）；要约束 2/3 方向须勾选并给极大刚度值。

## 坑与注意
- **阻尼必填频率**：瑞利阻尼靠"阻尼比+频率"反算，结构与土体频率取法不同（纯结构第一模态 vs 土体第一模态），取错则阻尼失真。
- **质量缩放门槛 10%**：目标时间增量是动力迭代步长，必须 ≤ 各单元稳定步长，否则质量缩放超 10% 结果失真——这是写出 inp 闪退（3.6f）、跨软件失败（3.8.3）、发散的高频根因。
- **人工边界（[[粘弹性人工边界]]）必须用表面集**：误用节点集等不当集合会导致写 inp 闪退（3.6g）。
- **人工边界 inp 在 ABAQUS 计算**（3.9）：写 inp 时勾"高级→写出 Vuel"，会多出人工边界单元并生成 `*.vload` 文件，需配套子程序（官网下载或找技术支持）处理；勾了 Vuel 的 inp 无法在 GFE 内计算，二者不可兼得。
- **批处理缺 inpx**（4.1）：批处理命令提交时 inp 路径若缺 `inpx` 文件，会丢失构件偏心、地震荷载、人工边界、工况、空间分布函数、包络输出等信息。
- **跨软件（ABAQUS，文中"软件 A"）差异**（3.8）：GFE 把 YJK 刚性杆转 SPRING2，显式动力不支持→须改 SPRINGA；桩-筏 embedded 嵌固在 ABAQUS 不支持→导入时填非零"桩合并阈值"改为共节点；场输出 NFORC1/NFORC2 须改为 NFORC（手册原文印作"NFROC"疑笔误，需实测确认）。
- **桩-土嵌固模态异常**（3.19）："线"嵌入"体"无法约束转动自由度，桩转角失真；须让桩与底板共节点或嵌入。
- **动接静前几秒震荡**（4.12）：动力步重力为瞬时加载，"质量-弹簧-阻尼"体系必然震荡；缓加重力或加大阻尼/弹簧刚度可减弱，显隐式单元转变也带来震荡（与 [[质量缩放]] 一致）。
- **覆土荷载重复计入**（4.14）：顶板覆土已由土自重形成，切勿再加附加质量/压力。
- **Tie 主从面**（4.13）：三维模型土做主面（粗网格主面）；二维平面应变结构做主面（细网格主面）。
- **闪退/中止杂项**：账户中文名/路径含空格或中文（3.7/3.11/3.14）、模型未划网格写 inp（3.6）、Office/WPS 版本（4.4）、未提示 success 就看结果（5.4）。
- **二维网格类型**（3.15）：反应位移法/反应加速度法用 B21、CPE3、CPE4R 等平面应变单元。
